# Components

Detail reference for `src/components/` (task, canvas, overlays, shared, layout, settings, taskboard) and `src/views/` helpers. Load when touching UI, canvas nodes, overlays, editors, or view-level grouping/bucketing.

## Views

| Abstraction | Location | Purpose |
|-------------|----------|---------|
| DashboardView | views/DashboardView.tsx | Horizon ribbon + two-column top row (Taskboard + hero horizon card, `@dnd-kit/sortable` order persisted via `settings.dashboardTopOrder`) + user-lists grid driven by `settings.dashboardUserLists` (sortable, per-card ⋯ overflow menu, "Add list" tile → `ListDefinitionPickerPopup`). Uses `buildDashboardLists` for every card. `DashboardListCard` is an internal helper in this file — not exported |
| truncateSections | views/ListView.tsx | Pure helper: walks ordered `Section[]` and slices the tail so total task count ≤ `maxTasks`; returns `{ displaySections, truncatedCount }`. Drives ListView's "hard" limit mode |
| buildDateSections, buildScheduledSections, buildDeadlineSections | views/ListView.tsx | Bucket todos by their date field into Overdue / Today / This Week / Later / (No Date\|Not Scheduled\|No Deadline). `buildDateSections` reads `effectiveDate(todo, today)`; `buildScheduledSections` reads `resolveScheduled(todo.scheduledDate, today)`; `buildDeadlineSections` reads `todo.dueDate`. Within a bucket order is `sortOrder` |

| CalendarOrientationToggle | components/canvas/rails/calendar/CalendarOrientationToggle.tsx | Shared ☰/☷ toggle button mounted in `WidgetHeader`'s `meta` slot by both `CalendarSlotContent` (wired to `canvas-rails-store.setSlotOrientation`) and `FloatingCalendarNode` (wired to `floating-calendar-store.updateOrientation`) |
| TabStrip | components/canvas/rails/TabStrip.tsx | Multi-tab slot header rendered by `RailsFrame.SlotRenderer` in place of `SlotHeader` when `slot.tabs.length >= 2`. Each tab pill shows `KIND_ICON` + label (lens → list name, others → kind label; pills grow to fit label width — no `max-width` cap — and still `min-width: 0` + `.label` `overflow:hidden; text-overflow: ellipsis` so the strip shrinks to an ellipsis when cramped; full label is the `title` tooltip) + `×` close; the active pill also carries a `⋯` menu (Pop out / Change type…). Trailing `+` button opens `WidgetKindMenu` to append a tab; trailing chrome buttons (pop-out ↗ / more ⋯ / close ×) mirror single-tab `SlotHeader`. A `⋮⋮` drag handle surfaces the whole-slot drag activator from `DraggableSlot`. Drag: pills are `useDraggable({ data: { kind: 'tab', slotId, tabId } })`; the strip is `useDroppable` on `encodeRailsDropId({ kind: 'tab-strip', slotId })` — drop onto another `TabStrip` reorders/merges via `reorderTab`/`moveTabToSlot`; drop onto a slot-level zone detaches + docks via `detachTabToNewSlot`. Keyboard: arrow-key roving (Left/Right/Home/End activates, Delete closes) + `role="tablist"` / `role="tab"` / `aria-selected` |
| RailsFrame | components/canvas/rails/RailsFrame.tsx | Outer rails shell. 3×3 CSS grid driven by `--left-size` / `--right-size` / `--top-size` / `--bottom-size` (0 when a rail is null); each rail is a grid item placed via `computeRailGridArea(rails, side)` so horizontal rails can extend into NW/NE/SW/SE corner cells when the corresponding `rails.corners[c] === 'h'`. `canvas-host` always occupies cell 2/2. Renders `DockOverlay` on top while a rails drag is in flight. Corner ownership is written at drop-time only (via `applyEmptySideCorners`) — there is no per-corner toggle UI. Hosts `useDefaultRails` (hydrates from `settings.canvasRails`, seeds a right-side `thisweek` lens on first run, debounce-persists back) and `useRailsDragMonitor` (routes `onDragEnd` to the matching store reducer + `applyEmptySideCorners(side, claim)` to wire the adjacent corners per the claimed/pinched semantics documented on `DockOverlay`). Also exports `popSlotToCanvas` / `popTabToCanvas` |
| DockOverlay | components/canvas/rails/DockOverlay.tsx | Drag-time drop-strip overlay. Renders only when the user is dragging a rails slot/tab. For each empty side, emits three `useDroppable` sub-zones — `start` / `center` / `end` — encoded via `encodeRailsDropId({ kind: 'empty-side', side, claim? })`. `start` / `end` extend into the adjacent corner cell (size keyed off the perpendicular rail's `--*-size` var so they collapse to 0 when that rail is absent). `useRailsDragMonitor.applyEmptySideCorners` resolves the drop with two roles per adjacent corner: **claimed** (dropped rail extends into the corner → owner = dropped-rail axis: `'h'` for top/bottom, `'v'` for left/right) and **pinched** (dropped rail does NOT extend → owner = opposite axis, so perpendicular rails own when present, with `resolveCorner` falling back cleanly when absent). `claim='start'` claims start + pinches end; `claim='end'` mirrors; no claim pinches both (the dropped rail is pinched between perpendicular neighbors). Writes route through `setCornerOwner` when the target ≠ default (`'v'`) and `clearCornerOwner` when it ==  default, keeping the persisted bag minimal. The strip wrapper is `display: contents` so each sub-zone positions absolutely against `.overlay` (a grid item spanning all 3×3 cells) rather than a 0-sized parent |
| RailContainer | components/canvas/rails/RailContainer.tsx | Rail shell. Renders an `<aside>` sized by the current `railSize` (width for verticals, height for horizontals; clamped to `[RAIL_SIZE_MIN=60, RAIL_SIZE_MAX=600]`). Two pointer-driven edge controls share a single rAF scheduler (`useThrottledResize`): `RailEdgeStrip` — an invisible 6 px drag strip spanning the full canvas-facing edge (drag-only, `col-resize` / `row-resize` cursor) — and `RailEdgeHandle` — a small neutral button (14×36 chip for vertical rails, 36×14 for horizontal) centered on the edge that click-toggles `toggleRailCollapsed(side)` and drag-resizes via `setRailSize` once the pointer has travelled past `DRAG_THRESHOLD_PX` (3 px). The handle uses `--color-surface` + `--color-border` + `--radius-sm` so it matches the rest of the app's chrome, hover brightens to `--color-surface-high`. Keyboard on the handle: Enter/Space toggle, arrow keys resize by 20 px. Collapse is an explicit action (button, keyboard) — dragging past the clamp minimum does not auto-collapse. When `collapsed === true` (from `rails.collapsed[side]`), the body content + edge strip are hidden (only the handle remains, rendered via `overflow: visible` so its half-outside chip stays fully visible against the narrow strip), and `railSize` returns `COLLAPSED_RAIL_PX` (28 px). The collapsed body renders one `CollapsedSlotStub` per slot — each shows the slot's `KIND_ICON` plus a semibold text label (the list definition name for lens slots via `useListDefinitionStore`, else the kind's title-cased `KIND_DISPLAY` label); labels are rotated with `writing-mode: vertical-rl` on vertical rails so they read top-to-bottom and stay horizontal on top/bottom rails |
