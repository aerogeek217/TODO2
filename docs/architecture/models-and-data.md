# Models & Data Layer

Detail reference for `src/models/` (TypeScript interfaces) and `src/data/` (Dexie persistence). Load when touching schema, repositories, migrations, import/export, or model shapes.

## Models

| Canvas | models/canvas.ts | Named spatial workspace |
| Project | models/project.ts | Positioned group of tasks on a canvas (optional color). Body content is derived from `todo-events` + `todo-store` via `selectDisciplineMetrics` — placement-only. Dexie secondary index on `canvasId` (table added in v44) |
| FloatingSnoozeGraveyard | models/floating-snooze-graveyard.ts | Canvas-pinned placement that renders the snooze graveyard (top-N most-rescheduled open todos) widget (stats-widgets-2026-04-25 P1). `{canvasId, x, y, width, height, collapsed?}`. Body content is derived from `todo-events` + `todo-store` via `selectMostDeferred` — placement-only. Dexie secondary index on `canvasId` (table added in v45) |
| TodoEvent, PersistedTodoEvent, TodoEventType | models/todo-event.ts | Append-only history log entry (stats-widgets-2026-04-25 P3). One row per tracked-field mutation on a todo (`scheduledDate` / `dueDate` / `statusId` / `isCompleted`) plus a `created` event at insert time. `TodoEventType ∈ {'created' \| 'scheduled' \| 'deadline' \| 'status' \| 'completed' \| 'reopened'}`. `fromValue` / `toValue` encoding: ISO date string for fixed dates / `'fuzzy:<token>'` for `ScheduledValue.kind === 'fuzzy'` / numeric `statusId` for status / null for unset / null on both sides for `created` / `completed` / `reopened` (the type carries the meaning). | Taskboard | models/taskboard.ts | **Singleton** queue of tasks: `{entries: TaskboardEntry[], createdAt, updatedAt}`. Referenced by the dashboard card, every rail-slot taskboard tab, and every `FloatingTaskboard` widget, so the same board renders live across all surfaces |
| TaskboardEntry | models/taskboard-entry.ts | Ordered slot within the singleton `Taskboard`: `{todoId, sortOrder}`. Stored inline on the Taskboard row — `todoId` uniquely locates an entry |
| Slot, Tab, TabType, SlotKind, Rail, RailsState, Corner, CornerOwner | models/canvas-rails.ts | Canvas-side-rail shapes. `SlotKind ∈ {'lens','notes','calendar','taskboard','horizons'}` (canonical `SLOT_KINDS`); `TabType` is an alias used by per-tab content type. `Slot` is `{ id, flex?, tabs: Tab[], activeTabId, orientation?, weekOffset? }` — every slot carries ≥1 `Tab { id, type: TabType, listDefinitionId?, runtimeFilterValue?: number[] }` (lens tabs only — `runtimeFilterValue` is the OR-combined picks for a def with `ListDefinition.runtimeFilter`). `getActiveTab(slot)` resolves the active tab (falls back to `tabs[0]` if `activeTabId` is stale). `RailsState` has per-side `Rail | null`, optional `widths`/`heights` bags (persisted even when the rail is null), and optional `corners: Partial<Record<Corner, CornerOwner>>` where `Corner ∈ {nw,ne,sw,se}` and `CornerOwner ∈ {'h','v'}` (`'h'` lets the top/bottom rail extend into the corner cell; default/absent `'v'` keeps the vertical-owns layout). `resolveCorner(rails, corner)` falls back to the orthogonal side when the stored owner's rail is absent; `computeRailGridArea(rails, side)` derives each rail's grid-area in the `RailsFrame` 3×3 grid. `cornerForSideClaim(side, 'start'\|'end')` maps an empty-side drop-strip sub-zone to the corner the new rail would claim. Helpers: `clampRailSize` ([200, 600]), `defaultRailSize`, `railSize`, `railOrientationForSide`, `EMPTY_RAILS`, `CORNERS`, `serializeRailsState` / `parseRailsState` (`parseSlot` requires `tabs[]` + `activeTabId`; `parseTab` accepts only the array shape for `runtimeFilterValue`; round-trips `corners`, dropping unknown keys and non-`'h'|'v'` values; returns null on shape failure) |
| Backup | models/backup.ts | Auto-snapshot record: trigger type, serialized data, size |
| ContextMenuItem | models/context-menu.ts | Shared shape for one row in a `<CanvasContextMenu>` panel: `{label, action, danger?, separator?}`. Hoisted in code-review-2026-04-30 P4 from the prior co-location inside `components/overlays/CanvasContextMenu.tsx` because `hooks/use-task-row-actions.ts` (a hook) imported it upward against the layer rule. | AppView | models/app-view.ts | Enum: Canvas, Dashboard, List, Calendar, Settings |
| ListSortBy | models/app-view.ts | Type alias over `TodoSortBy` (ui-consistency-2026-04-25 P4 unified the per-surface sort enums). | ListGroupBy | models/app-view.ts | Type alias over `TodoGroupBy` (ui-consistency-2026-04-25 P4). Drives `ListView` section-builder choice; `'none'` renders a flat list (header hidden). `applyDefinition` coerces an incoming `'name'` grouping (defensive — `'name'` is sort-only) to `'none'` |
| ListItemSortBy | models/app-view.ts | Type alias over `TodoSortBy` (ui-consistency-2026-04-25 P4). ListView constrains its visible options via `LIST_SORT_VALUES` from `models/todo-sort-group.ts`. | DateField | models/app-view.ts | Type: date, scheduled, deadline, created, modified — used by filter store and saved views. `'date'` filters on `effectiveDate`; `'scheduled'` on resolved `scheduledDate`; `'deadline'` on `dueDate` |
