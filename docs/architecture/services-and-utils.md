# Services, Hooks & Utils

Detail reference for `src/services/` (non-UI logic), `src/hooks/` (custom React hooks), and `src/utils/` (pure utilities). Load when touching NLP, recurrence, file storage, dashboard list interpretation, drag-and-drop logic, keyboard shortcuts, or date helpers.

## Services

| Abstraction | Location | Purpose |
|-------------|----------|---------|
| buildDashboardLists, interpretMembership, interpretSort, interpretGrouping, WARNING_WINDOW_DAYS | services/dashboard-lists.ts | Pure interpreter over `ListDefinition`. Membership kinds: today (≤ today OR deadline within `warningWindowDays`, default WARNING_WINDOW_DAYS=3), upcoming (has date, not in today), deadlines (has `dueDate`, intentional overlap with today), someday (no dates), custom (injected `ctx.evalPredicate` consumes a `TodoPredicate`). Sort kinds: effective-date-asc (expired-fuzzy first), deadline-asc, sort-order, sortBy (chronological `by` values sort properly; categorical fall back to sortOrder). Grouping kinds: none, relative-effective, relative-deadline, by-sortBy (`'by-tag'` reads `ctx.assignedTagsMap` → registry rows for bucket label + color; parallels `assignedPeopleMap` / `assignedOrgsMap`; a todo with multiple tag assignments appears in every matching bucket). Applies `showCompleted` + `showHiddenStatuses` gates. Date-based sort + bucket compare by each todo's own date with id as a stable tiebreaker |
| buildExportData | services/export-import.ts | Reads all DB tables (incl. `listDefinitions`) in parallel; shared by file-storage, settings export, and backup snapshots |
| buildMarkdownExport | services/export-import.ts | Builds markdown representation of all tasks grouped by project; flat list per project (no parent/child nesting); shows `[status.name]` for meaningful statuses (icon or hideByDefault); uses buildExportData |
| fileStorageService | services/file-storage.ts | File System Access API sync (file ↔ IndexedDB); uses onAfterImport callback for store refresh; `onConfirmMigration` callback pauses import of legacy-format files pending user confirmation |
| backupScheduler | services/backup-scheduler.ts | Auto-snapshot every 24h, pre-destructive snapshots, prune to 10 max; started in App.tsx |
| checkMigrationNeeded | services/migration-check.ts | Checks IndexedDB version via `indexedDB.databases()` before Dexie opens; converts IDB version (Dexie multiplies by 10) to Dexie version for comparison; returns `MigrationInfo` if data-modifying upgrade is pending |
| detectLegacyFormat | services/migration-check.ts | Inspects raw parsed JSON for legacy fields (`isStarred`/`isAssigned` booleans, `starred` list insets, `priority`/`isHardDeadline` todo fields, `high-priority`/priority-`attributeFilter` list insets); returns `LegacyImportInfo` with per-category counts and human-readable descriptions |
| exportCurrentDatabase | services/migration-check.ts | Reads all tables from raw IndexedDB at a specified version (without triggering Dexie upgrade); returns JSON string |
| undoable | services/undoable.ts | Helper to register an action as undoable; skips when undo store is mid-undo/redo |
| task-placement | services/task-placement.ts | Pure functions for task ordering: computeInsertionSort, placeTaskAt, placeMultipleAt, normalizeSortOrders, shouldNormalize |
| pasteTasksAt | services/clipboard.ts | Paste cut tasks at a target position using placeMultipleAt + applyMutations; clears clipboard after paste |
| drop-resolver | services/drop-resolver.ts | Pure drop target resolution: resolveDropTarget (DropResolution), resolveDropPreview (preview indicators) |
| parseInput | services/natural-language-parser.ts | Parses raw text for NLP tokens: `@person` or `@"First Last"`, `/project`, `#tag` (whitespace-anchored; group-1 separator stays in the remaining title; slug is `normalizeTag`-folded in first-seen order), date keywords (→ `scheduledDate`), deadline syntax `by <date>` / `!<day>` (→ `dueDate`, fuzzy windows resolve to end-of-window), recurrence (`every week`, `every quarter`, `repeat daily`); returns ParsedInput with cleaned title, persons[], projects[], tags[] (string slugs — `resolveTags` turns them into registry ids downstream), scheduledDate, dueDate, recurrence |
| makeRecurrenceRule | services/recurrence.ts | Build a RecurrenceRule, capturing originalDayOfMonth for monthly/quarterly/yearly |
| computeNextDueDate | services/recurrence.ts | Advances a due date by one recurrence interval, skipping past dates |
| recurrenceAnchor, advanceRecurring | services/recurrence.ts | `recurrenceAnchor(todo)` picks `dueDate` first, falls back to a precise `scheduledDate` (returns null for fuzzy-only). `advanceRecurring(todo)` computes the next-occurrence field update — either `{dueDate}` or `{scheduledDate: {kind:'date', value}}` — used by `todo-store.toggleComplete` / `bulkSetCompleted` and CalendarView virtual instances |
| generateRecurringInstances | services/recurrence.ts | Generates all recurring dates within a date range for calendar display |
| resolveInput | services/nlp-resolver.ts | Matches parsed person/project/org names against known entities (case-insensitive exact/prefix/initials/first-name); person-first for @tokens, unmatched fall through to org matching; returns personIds[], orgIds[], projectId, unmatched names, plus the pass-through `tags: string[]` from `ParsedInput` (converted to ids by `resolveTags`) |
| resolveTags | services/nlp-resolver.ts | Async resolve-or-create: for each `#foo` slug, look up the existing tag case-insensitively in `ctx.tagStore.tags` and return its id; on a miss, `tagStore.add(name, DEFAULT_ENTITY_COLOR)` and use the new id. Returns ids in input order, deduped. A local `createdByLower` cache coalesces duplicate new slugs within the same call so `#Foo #foo` creates one tag, not two |
| parseTaskInput | services/nlp-task-creator.ts | Combines parseInput + resolveInput; `applyNlpMetadata` assigns parsed metadata (people, orgs) inside `runNlpMetadataTransaction`, then calls `resolveTags` against `useTagStore` and assigns each resolved tag id via `tag-store.assignTag` — tag writes live outside the transaction because the registry rows are normalized and don't need atomic ordering with person/org assignment |
| convertFloatingKind | services/float-kind-switch.ts | Deletes the source float record and creates a fresh float of the target kind at the same `{x, y, width, height}`. Resolves `listDefinitionId` (lens) from caller seed or the list-definition store; taskboard targets need no resolver because the board is a singleton. Aborts and leaves the source intact when no list-def is available for a lens target. Used by every floating widget's title-menu kind-change path |
| saveFileHandle, loadFileHandle, clearFileHandle, saveLastPickerHandle, loadLastPickerHandle | services/file-handle-idb.ts | File System Access API handle persistence in a dedicated `todo2-meta` IndexedDB (separate from the main Dexie DB so legacy-import "clear all tables" won't drop the handle). Handles stored under namespaced keys (`todo2-primary`, `todo2-last-directory`); read paths transparently migrate legacy un-namespaced keys forward on first access |
| horizons (HORIZON_KEYS, HorizonKey, ribbon bin geometry) | services/horizons.ts | Canonical horizon definitions used by the dashboard ribbon + `parseHorizonSlots` |
| notes-export (mdToHtml, htmlToMarkdown, copyNotesRich, sanitizeHref) | services/notes-export.ts | Markdown ⇄ HTML clipboard helpers for the notes toolbar's rich-copy action. `sanitizeHref` uses an allowlist (http/https/mailto/tel + schemeless) to neutralize `javascript:` / `data:` / `vbscript:` / `file:` / `blob:` URLs in both directions; Phase 0 security hotfix |

| useNlpAutocomplete | hooks/use-nlp-autocomplete.ts | Hook for `@` / `/` / `#` autocomplete in input fields: tracks trigger position (whitespace-anchored), filters people/orgs/projects/tags (case-insensitive prefix match for `#`), handles arrow/Tab/Enter/Escape navigation. ## Utils

| Abstraction | Location | Purpose |
|-------------|----------|---------|
| resolveFuzzy, resolveScheduled, effectiveDate, isScheduledExpired, isScheduledPast, isDeadlinePast, scheduledLabel, scheduledValuesEqual, daysUntil, dateIntensity, setConfiguredWeekStart, getConfiguredWeekStart | utils/effective-date.ts | Unified scheduling helpers: resolve `ScheduledValue` fuzzy tokens to concrete end-of-window dates, compute `min(scheduled, deadline)`, label chips, structural equality for ScheduledValue. `daysUntil(date, today)` returns whole-day diff; `dateIntensity(days)` returns a 0.15..1 proximity factor used by task-row chips to fade toward `--color-text-muted` for distant dates. `isScheduledExpired` is fuzzy-only; `isScheduledPast` broadens to fuzzy-expired OR precise past; `isDeadlinePast` covers `dueDate < today`. Every sort/filter/group consumer reads `effectiveDate`. `resolveFuzzy(token, today, weekStartsOn?)` takes an optional week-start override (0=Sunday, 1=Monday); default is the module-level value managed by `setConfiguredWeekStart`, which settings-store keeps in sync with the persisted `weekStartsOn` setting |
| MS_PER_DAY, startOfDay, startOfToday, isSameDay, formatDate, formatRelativeTime, toDateInputValue | utils/date.ts | Centralized date utilities: day normalization, formatting, constants |
| bySortOrder | utils/sort-order.ts | Shared sort comparator: sortOrder ascending, with id as a stable tiebreaker so equal-sortOrder tasks render in deterministic order |
| generateInitials | utils/person.ts | Generates 1-3 character uppercase initials from a name |
| resolvePersonColor | utils/person-color.ts | Returns the first assigned org's color for a person (`personId + personOrgMap + orgs`), or `undefined` when no org is assigned / has a color. Replaces the retired `Person.color` field (dropped in Dexie v31); used by `AvatarStack`, `TaskEditMetadata`, `ListView`, and NLP autocomplete pipelines |
| toggleItem, matchTodoText, TextMatchContext, TextMatchField, TEXT_MATCH_FIELDS | utils/filter.ts | `toggleItem` toggles an item in a null-or-Set filter (null = all shown, Set = explicit selection). `matchTodoText(todo, query, ctx)` is the canonical multi-field text matcher used by TopBar search + `filter-store.matchesFilter` — hits title/notes from the todo itself and project/person/org/status/tag from `TextMatchContext` when the caller supplies them. Callers resolve `tagNames` from `useTagStore().assignedTagsMap.get(todoId)?.map(t => t.name)` (the registry), never from the retired inline `todo.tags` field |
| normalizeTag, TAG_SLUG_RE, TAG_MAX_LEN | utils/tags.ts | Pure slug normalizer for `#foo` tokens: trims, lowercases, validates against `/^[a-z0-9_-]+$/`, caps at 64 chars. Returns `null` for anything that can't round-trip through the NLP. Shared by the natural-language parser + import validation; the normalized slug feeds `nlp-resolver.resolveTags` which maps it to a registry id |
| getFilterDefaults | utils/filter-defaults.ts | Extract task creation defaults (people, orgs, status, project) from active filter criteria; strips sentinel 0 values; single-selection rule for status + project |
| supplementWithFilterDefaults | utils/filter-defaults.ts | Supplement resolved NLP output with filter-inferred defaults (person/org/project); mutates resolved in place (void return) |
| isMacLike, formatShortcut | utils/platform.ts | Platform detection + keyboard-shortcut label formatting. `isMacLike()` prefers `navigator.userAgentData.platform` and falls back to `navigator.platform`. `formatShortcut('Mod-…'/'Alt-…'/'Shift-…')` renders CodeMirror-style specs as `⌘`/`⇧`/`⌥` on Mac-like platforms and `Ctrl+`/`Shift+`/`Alt+` elsewhere; used by NotesBody footer + toolbar and any UI that surfaces a shortcut hint |
| KIND_ICON, KIND_LABEL | utils/slot-kind.ts | Per-`SlotKind` icon-glyph + label maps shared by `WidgetHeader`, `WidgetKindMenu`, `SlotHeader`, and floating nodes so every surface renders the same kind affordance |
| TASK_DRAG_KIND, TASK_DROP_KIND, isTaskDragKind | utils/task-dnd/kinds.ts | Canonical `data.type` strings for every task-shaped drag payload + drop target. `TASK_DRAG_KIND = { task, taskboardTask }` collapsed the three pre-unification aliases (`task` / `list-task` / `dashboard-task`) into a single `'task'` kind; `'taskboard-task'` stays separate because its drop-off-target behavior (remove from board) differs. `TASK_DROP_KIND = { task, project, taskboard, taskboardTask, calendarDay, listSection }` is what drop handlers match on via `event.over.data.current.type`. `isTaskDragKind(v)` narrows a `data.type` value to the drag-payload subset |
| taskDragId, isTaskDragId, parseTaskboardEntryId, projectDropId, taskboardFloatDropId, calendarDayDropId, TASKBOARD_SINGLETON_DROP_ID | utils/task-dnd/ids.ts | Central id factories for every draggable-task surface. `taskDragId(surface, todoId, extras?)` emits the per-surface drag id — `canvas-project → todo-<id>` / `inset → inset-todo-<id>` / `lens → lens-todo-<id>` / `list → list-todo-<id>` / `dashboard → dashboard-<listKey>-<id>` / `taskboard-panel → tbp-<id>` / `taskboard-float → tb-<floatingId>-<id>` / `calendar-view → calview-todo-<id>` / `calendar-strip → calstrip-todo-<id>`; throws when required extras are missing. `parseTaskboardEntryId(id)` replaces the fragile `Number(id.split('-').pop())` sniff (F9 in dnd-audit) with an explicit `tbp-<todoId>` / `tb-<floatingId>-<todoId>` parser used by `dispatchTaskDrop`. `projectDropId`/`taskboardFloatDropId`/`calendarDayDropId(scope, dateMs)` emit the matching drop-target ids; `TASKBOARD_SINGLETON_DROP_ID` is the hard-coded dashboard/rail singleton droppable id |
| buildTaskCollision | utils/task-dnd/collision.ts | Shared `CollisionDetection` factory. Takes a declarative `TaskCollisionRule[]` (each rule = `{ when, accept, algorithm: 'pointerWithin' \| 'closestCenter' \| 'rectIntersection' }`) — the first `when`-match wins and its `accept` filter scopes the droppable set before the algorithm runs. Consumed by `CanvasPage` (rails vs. non-rails — F12 isolation: rails drags match `rails:*` zones, everything else gets the non-rails zones), `DashboardView` (top-row / user-list / taskboard), and `CalendarView` (day cells only). Falls back to `closestCenter` when no rule matches or `active` is null |
| dispatchTaskDrop, TaskboardOps, CalendarOps | utils/task-dnd/dispatch.ts | Shared drop dispatcher consumed by `use-canvas-dnd.ts` + `DashboardView.handleDragEnd` + `CalendarView`. Owns the four taskboard/calendar branches: (1) `calendarDay` target → `calendar.reschedule(todoId, date)` — checked first so taskboard entries dragged onto the mini-calendar reschedule instead of removing from the board; (2) `taskboard-task` drag onto a `taskboard-task` target → `reorder(from, to)`; (3) `taskboard-task` drag onto the panel container → move to end; (4) `taskboard-task` drag with `over=null` (or non-taskboard target) → `removeEntry`; plus external `task`-kind drag onto the board → `addAt` / `addMultipleAt` at the index implied by dnd-kit's sortable data. Returns `false` when the drop isn't taskboard/calendar-shaped so the caller's route-specific handler (e.g. canvas project placement) can run. `TaskboardOps` is the narrow singleton-board surface `useTaskboardStore.getState()` implements directly; `CalendarOps` is a single `reschedule` callback |
| RAILS_DRAG_TYPE, RailsDragData, SplitZone, TabDropTarget, encodeRailsDropId, decodeRailsDropId, applyDropToSide, applySplitDrop, applySplitButton, applyReorderTab, applyMoveTabToSlot, applyDetachTabToNewSlot | utils/rail-dnd.ts | Pure reducers + drop-id codec for rail drag-and-drop. `RailsDragData` discriminates on `kind: 'slot' \| 'tab'`; `tab` drags carry `sourceSlotId` + `tabId`. Drop zones include the existing side/quadrant/split kinds plus a `tab-strip` zone (pill-reorder / merge-into-slot). `applyReorderTab` + `applyMoveTabToSlot` handle tab-strip drops; `applyDetachTabToNewSlot` extracts a tab into a fresh single-tab slot and reuses `applyDropToSide`/`applySplitDrop` to dock it. Consumed by `useCanvasRailsStore` and the rail components (Phase 2 relocated this out of `components/canvas/rails/`) |
| isValidCssColor | utils/css.ts | Validates hex color strings (#rgb or #rrggbb only); consumed by settings-store color overrides + import validation |
| parseHorizonSlots | utils/horizon-slots.ts | Parses `settings.horizonSlots` JSON into `Partial<Record<HorizonKey, number>>`. Invalid slot keys silently dropped; never throws |
